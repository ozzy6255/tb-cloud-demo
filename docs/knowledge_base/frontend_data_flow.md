# 前端数据流：本地计算 vs 服务端计算

## 1. 您的数据从哪来？
在当前的 "Vue + FastAPI" 架构中，数据经历了以下旅程：
1.  **源头**: SQLite 数据库文件 (`app.db`)。
2.  **搬运**: Python 后端 (`main.py`) 从数据库读出 225 条数据。
3.  **运输**: 通过 HTTP API (`/products`) 发送到浏览器。
4.  **存储**: 浏览器把這 225 条数据暂时存到了**内存**里 (Vue 的 `products` 变量)。

## 2. 搜索为什么这么快？(Client-Side Search)
您提到的“即时搜索”之所以感觉不到延迟，是因为：
*   **搜索动作发生在“本地”**: 当您打字时，Vue 并没有去问后端，而是直接在**浏览器的内存**里筛选那 225 条数据。
*   **术语**: 这叫 **Client-Side Filtering (客户端过滤)**。
*   **适用场景**: 数据量小 (几千条以内)。体验极佳。
*   **不适用场景**: 数据量巨大 (比如淘宝几亿商品)。那时就必须每打一个字都去问一次服务器 (**Server-Side Search**)。

## 总结
**数据来自数据库，但搜索计算发生在您的浏览器里。**
